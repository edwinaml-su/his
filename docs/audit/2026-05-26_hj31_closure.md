## Hallazgo

**HJ-31 (audit Stream J, 2026-05-19) — P1 ALTA**

> Merge-queue: sin verificación server-side de quorum de roles. La UI muestra dos campos PIN (Director + Director Médico) asumiendo dos personas distintas, pero `patientDedup.confirmEceMerge` podría aceptar dos firmas del mismo usuario con el mismo rol — un único Director podría auto-aprobar fusiones irreversibles.

---

## Resolución

**Cerrado en:** PR #226 (`33b7910`) — 2026-05-24
**Mecanismo:** Helper `assertQuorumOrThrow` en `packages/trpc/src/routers/patient-dedup.router.ts:119-171`, invocado desde `confirmEceMerge:455` antes del UPDATE.

### Validaciones server-side

El helper aplica **3 reglas** consecutivas, en orden:

1. **Personal distintos** (línea 125): si `firma1.personal_id === firma2.personal_id` → `TRPCError code='FORBIDDEN'` con mensaje *"Quorum inválido: ambas firmas corresponden al mismo profesional."*
2. **Rol ECE vigente requerido** (línea 158): si cualquiera de los dos firmantes no tiene un rol ECE activo en `ece.asignacion_rol` → `TRPCError code='PRECONDITION_FAILED'`.
3. **Roles distintos** (línea 165): si `rol1 === rol2` → `TRPCError code='FORBIDDEN'` con mensaje *"Quorum inválido: ambos firmantes tienen el mismo rol ECE ({rol}). Se requieren roles distintos."*

### Capa adicional

- La procedure se declara como `requireRole(["DIR", "ADMIN"])` (línea 417) — solo Director o admin de tenant puede invocar el endpoint.
- Los PINes se verifican con argon2id server-side antes del quorum check (HJ-30, líneas 446-452).
- El audit log persiste `firmaDir1Id` + `firmaDir1PersonalId` + `firmaDir2Id` + `firmaDir2PersonalId` (líneas 502-505) — trazabilidad completa sin filtrar PINes.

## Diferencia vs especificación original

El audit recomendaba validar **roles específicos** (`firmante1 = DIR`, `firmante2 = DIR_MEDICO`). El código actual implementa una versión **más flexible**: cualquier par de roles ECE distintos del directorio (`ece.rol`). Justificación operativa:

1. La procedure ya está limitada al rol caller `["DIR", "ADMIN"]` (`requireRole`).
2. El quorum "roles distintos + personas distintas" cumple el objetivo regulatorio del audit (prevenir auto-aprobación).
3. Permite escenarios legítimos donde un par {Director, Subdirector} o {Director, Director Médico} firmen — flexibilidad operativa sin sacrificar el principio de separación.

Si NTEC exige enforcement estricto de pareja `DIR + DIR_MEDICO`, se puede ajustar `assertQuorumOrThrow` para whitelist; el cambio es local y trivial.
